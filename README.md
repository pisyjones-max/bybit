# NOVATION Platform — Multi-user Quant Terminal (Bybit Spot)

Веб-приложение, где каждый пользователь регистрируется, вписывает
**свои** API-ключи Bybit, настраивает свои торговые параметры и
получает свой личный дашборд. Торговый цикл крутится в фоне на
сервере — не зависит от того, открыт ли у кого-то браузер.

## Как это устроено

- `models.py` — пользователи, зашифрованные API-ключи, персональные
  настройки, позиции и журнал сделок в базе (SQLite по умолчанию,
  легко переключить на Postgres через `DATABASE_URL`).
- `auth.py` — регистрация, вход, выход, страница настроек
  (`/settings`) для ввода ключей и параметров бота.
- `engine.py` — фоновый поток: раз в 3 секунды тянет цены (общий
  публичный запрос для всех), потом проходит по каждому активному
  пользователю и торгует **его собственными** ключами и параметрами.
- `dashboard.py` — Dash-дашборд, защищён логином, показывает только
  данные текущего пользователя.
- `app.py` — точка входа, связывает всё вместе.

## ⚠️ Важные правила эксплуатации

1. **Только ОДИН активный экземпляр движка.**
   `RUN_ENGINE=1` должен стоять ровно на одном запущенном процессе.
   Если поднимаешь несколько воркеров/инстансов — везде, кроме одного,
   ставь `RUN_ENGINE=0`. Иначе один и тот же сигнал обработается
   несколько раз и ордера продублируются для одних и тех же людей.
   `Procfile` уже настроен на `--workers 1` намеренно — не увеличивай.
2. **API-ключи каждого пользователя шифруются** (`MASTER_KEY`,
   Fernet) перед записью в базу. `MASTER_KEY` и `FLASK_SECRET_KEY`
   должны быть настоящими сгенерированными секретами в проде, не
   значениями из `.env.example`.
3. **Файл базы данных лежит в `instance/`**, а не в корне проекта —
   так работает Flask-SQLAlchemy с относительными sqlite-путями.
   Это не файл для гита — `.gitignore` его исключает, но не забудь
   **делать бэкапы `instance/novation.db`** (или сразу используй
   Postgres в проде — так надёжнее для 10-15 активных аккаунтов).
4. **Каждый человек торгует своими деньгами через свои ключи.** Бот
   не объединяет чужие средства — если кто-то не добавил ключи или
   выключил "бот активен" в настройках, для него ничего не
   происходит.
5. Считай сообщением о рисках: **бот реально исполняет ордера на
   бирже**. Не давай доступ людям, которые не понимают, что могут
   потерять деньги.

## Локальный запуск

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt

cp .env.example .env
# сгенерируй и впиши реальные секреты:
python3 -c "import secrets; print(secrets.token_hex(32))"                       # → FLASK_SECRET_KEY
python3 -c "from cryptography.fernet import Fernet; print(Fernet.generate_key().decode())"  # → MASTER_KEY

python3 app.py
```

Открой http://localhost:8000 — попадёшь на страницу входа/регистрации.

## Что видит пользователь

1. `/register` → создаёт аккаунт
2. `/settings` → вводит Bybit API key/secret, включает "бот активен",
   настраивает размер позиции, RSI, TP/SL, лимит позиций
3. `/dashboard/` → живой сканер рынка, график, журнал своих сделок

## Деплой на сервер

Подходит любой хостинг с постоянным процессом (не Vercel/serverless):

**Railway / Render**
- Подключи GitHub-репозиторий
- Build: `pip install -r requirements.txt`
- Start: команда уже в `Procfile`
- Переменные окружения: `FLASK_SECRET_KEY`, `MASTER_KEY`,
  `DATABASE_URL` (если Postgres), `RUN_ENGINE=1`

**Обычный VPS**
```bash
git clone <repo>
cd novation-platform
python3 -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt
cp .env.example .env   # впиши реальные секреты
gunicorn app:server --bind 0.0.0.0:8000 --workers 1 --threads 8
```
Запускай под `systemd` или `tmux`, чтобы процесс жил после закрытия SSH-сессии.
